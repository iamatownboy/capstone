## 1. 개요 및 취지
개발 생산성과 릴리즈 안정성을 높이기 위한 소스코드 버전 관리 및 브랜치 배포 전략입니다. 한빛클라우드 개발자들의 원활한 협업을 위한 Git 브랜치 전략 가이드와 코드 리뷰 지침입니다.

## 2. 주요 운영 가이드라인
우리는 Git Flow 기반의 변형 전략을 사용하며, feature 브랜치 작업 완료 후 Pull Request가 권장됩니다. 모든 코드는 최소 1명 이상의 피어 리뷰어 승인을 득한 후 master/main에 머지될 수 있습니다. 따라서 각 담당자는 본 가이드라인의 세부 사항을 명확히 인지하고, 실무 적용 시 발생할 수 있는 취약점을 사전 예방하는 데 만전을 기해주시기 바랍니다.

## 3. 상세 항목 및 절차
개발용 로컬 테스트 서버 IP는 192.0.2.120이며, CI/CD 빌드 상태는 사내 젠킨스 웹(dev.example.test)에서 실시간 확인됩니다. 의견 충돌이 있을 시에는 슬랙 채널(dev.example.test) 혹은 권한심사팀 개발 파트 미팅을 통해 조율하시기 바랍니다.

## 4. 유의사항
코드 퀄리티 유지와 오너십 공유를 위해 모든 커밋 메시지 컨벤션을 명확히 지켜주십시오. 조치 사항에 의문이 있으실 경우 권한심사팀 담당자(jian.oh15@example.test) 혹은 사내 포털 사이트(dev.example.test)를 통해 질의 바랍니다.
